# V3-2：一键草稿 / 一键正式发布 + 状态跳转 Tab

## 1. 目的与范围
- 同时支持“一键保存草稿”和“一键正式发布”两种动作。
- 当用户选择“直接发布”后，插件应尽最大可能自动完成发布链路，**无需再次确认**。
- 若部分渠道失败：面板显示失败状态；点击任一渠道的状态（成功/失败/未登录/等待处理）可跳转到对应浏览器 Tab。

## 2. 动作定义与成功标准
### 2.1 保存草稿（draft）
- 定义：在目标平台将内容保存为草稿/未发布状态。
- 成功标准：满足 `V2-0` 的“草稿成功可见标准”之一。

### 2.2 直接发布（publish）
- 定义：触发平台发布提交，并最终确认文章已发布（详情页/列表页可见或出现“发布成功”等提示）。
- 成功标准：满足 `V2-0` 的“发布成功可见标准”之一。

## 3. “无需确认”的含义（必须）
- 在 `publish` 动作下：
  - 若平台存在“发布”按钮与“确认发布/确定”二次确认按钮：插件应自动点击至完成。
  - 若平台出现必须由用户完成的步骤（登录/验证码/实名/风险提示/必填项）：允许进入 `waiting_user`，但不得要求用户再“确认发布动作”本身。

## 4. 面板状态与 Tab 跳转（必须）
### 4.1 状态枚举（面板展示）
- 未开始
- 进行中
- 成功
- 失败
- 等待处理
- 未登录

### 4.2 可点击跳转规则
- 点击任意状态 badge：
  1) 若该渠道 Tab 存在：聚焦该 Tab 并将其所在窗口置顶
  2) 若 Tab 已被关闭：重新打开该渠道创作入口页，并继续显示该渠道状态（便于用户处理登录/风控/必填项）

## 5. 验收标准
- “保存草稿/直接发布”两种动作均可在面板中一键发起并并发执行。
- 任一渠道执行完成后，面板可正确显示最终状态。
- 点击任一渠道的状态 badge，可稳定跳转到对应 Tab；Tab 不存在时能重开创作页。
