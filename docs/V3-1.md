# V3-1：图片自动上传（正文图片）

## 1. 目的与范围
- 目标：在“保存草稿/直接发布”两种动作下，**所有渠道**都应支持把微信公众号正文中的图片下载后上传到目标平台，并按原文顺序插入到正确位置。
- 本文只定义需求与验收口径，不包含实现代码细节。

## 2. 输入与输出
### 2.1 输入（来自微信抓取源）
- 标题（string）
- 正文 HTML（string）
- 原文链接（string）

### 2.2 输出（到各发布渠道）
- 目标平台编辑器中，图片不再依赖微信外链；由平台自身上传生成可用的图片资源。
- 图片顺序与原文一致；图片在正文中的相对位置与原文一致（段落级/行级尽量一致）。

## 3. 核心行为定义（需求级）
1) 识别正文中所有图片（`<img>`），优先使用 `data-src`，其次 `src`。
2) 下载图片二进制（仅在用户本地浏览器内进行，不上传到第三方服务器）。
3) 在目标平台编辑器中，以“粘贴/拖拽/文件 input 上传”等方式触发平台自身上传。
4) 插入位置规则：图片在微信正文中的出现顺序即为插入顺序；图片前后的文本段落相对顺序保持不变。

## 4. 诊断与提示（必须）
在发布面板诊断区/状态行中，至少提供：
- 当前处理进度：例如“上传图片（2/8）”
- 失败原因：下载失败 / 上传失败 / 编辑器不支持 / 风控/验证码等
- 建议操作：例如“请在该平台编辑器内手动上传该图片后点击继续/重试”

## 5. 常见失败场景（必须覆盖）
### 5.1 图片下载失败
- 可能原因：图片 URL 非法/不在允许域名范围/网络错误/图片过大
- 期望行为：该渠道进入 `failed` 或 `waiting_user`，诊断给出明确提示与原图 URL（开发态可展示更多细节）。

### 5.2 平台上传失败
- 可能原因：平台要求登录/实名/风控；编辑器禁用粘贴；需要手动点“插入图片”按钮
- 期望行为：进入 `waiting_user`，提示用户手动完成上传后点击“继续”。

### 5.3 图片位置错乱
- 期望行为：以“顺序正确”为底线；若平台强制把图片插入到光标末尾，应在诊断提示“该平台图片将按顺序插入，但可能无法做到段落级完全一致”。

## 6. 验收标准
- 任意包含图片的微信文章，在所有已勾选渠道执行后：
  - 正文中图片可见、可打开、非微信外链（以平台实际渲染为准）
  - 图片数量与原文一致（允许平台自动合并/压缩导致 URL/格式变化，但数量与顺序一致）
  - 图片前后的文字段落顺序不乱
