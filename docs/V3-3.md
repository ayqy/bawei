# V3-3：自动检测登录状态

## 1. 目的与范围
- 用户在微信文章页打开发布面板后，点击开始执行：
  - 插件一键打开所有已勾选渠道的“文章创作/编辑入口页”
  - 自动检测每个渠道是否已登录
  - 若未登录或登录失效：面板明确显示“未登录”，并允许点击状态跳转到对应 Tab

## 2. 登录检测触发时机（必须）
- 打开渠道入口页后立即检测（无需等待用户操作）。
- 若页面发生重定向（例如跳转到登录页/统一认证页）：应能识别并更新状态。

## 3. 登录失效的定义（需求级）
满足任一条件即判定为“未登录/登录失效”：
- URL 命中该渠道登录页/鉴权页/统一认证页的特征（例如 `login/signin/passport/oauth/auth`）
- 页面出现明显的登录表单/登录按钮/“请登录”等提示（以渠道页面实际 DOM 为准）

## 4. 面板展示与交互（必须）
- 渠道状态显示为：`未登录`
- 诊断区提示：需要登录后点击“重试/继续”
- 点击“未登录”状态可跳转到该渠道 Tab（或重开入口页），便于用户完成登录

## 5. 验收标准
- 在任意渠道未登录时：
  - 面板能显示“未登录”
  - 点击状态可打开/聚焦对应 Tab
  - 用户登录完成后点击“重试”可继续该渠道流程
